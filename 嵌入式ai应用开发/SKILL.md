# 核心开发原则
 
1. **小步快跑**：每次只实现一个功能模块，验证通过后再进行下一个
2. **禁止猜测**：所有模型选择、推理框架选择、性能优化方案必须先向我确认
3. **最小代码量**：只实现当前需求的最小功能集，不提前优化
4. **可验证交付**：每个功能模块都要有明确的性能指标和精度指标
5. 只有当用户明确输入 `@嵌入式Linux应用开发核心`时，本技能才会生效
# 技术栈要求
- 优先使用RKNN Toolkit2部署YOLO系列模型
- 推理框架优先使用ONNX Runtime、TensorRT Lite
- 编程语言优先使用C++，其次是Python
- 多线程处理优先使用std::thread和std::mutex
- 数据传输优先使用零拷贝技术

# 性能要求
- 明确标注每个推理步骤的耗时
- 内存使用量控制在硬件限制范围内
- CPU使用率不超过80%
- 帧率达到项目要求的最低标准

# 输出格式要求
1. 首先明确列出本次任务的**性能指标**和**精度指标**
2. 说明使用的模型版本和推理框架
3. 提供核心推理代码
4. 添加性能统计代码
5. 提供编译和运行命令
6. 列出需要我确认的重大决策（如模型选择、量化方式）

# 特殊要求
- 当需要选择模型时，提供不同模型的精度、速度、内存占用对比
- 当需要优化性能时，先进行性能分析，找出瓶颈，再提出优化方案
- 当模型精度不达标时，分析原因，提出可能的解决方案
- 提供模型转换和量化的详细步骤
