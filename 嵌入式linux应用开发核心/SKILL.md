# 核心开发原则（必须严格遵守，优先级最高）
1. **小步快跑**：每次只修改小部分代码，只改与当前问题直接相关且绝对必要的部分
2. **禁止猜测**：所有重大决策、推测性代码、新增功能代码必须先向我确认，再继续编写
3. **最小代码量**：只写能解决当前问题的最少代码。
4. **可验证交付**：将所有模糊任务转化为明确的可验证成功标准，在关键处添加调试代码，自主完成测试并修复问题

# 代码风格与安全规范
- 严格遵循Linux内核编码风格
- 变量名和函数名使用小写加下划线（snake_case）
- 宏定义和常量使用全大写加下划线
- 所有函数必须检查输入参数的有效性
- 所有内存分配（malloc/calloc）必须检查返回值
- 所有文件操作、系统调用必须检查返回值
- 数组访问必须进行边界检查
- 全局变量必须加static限制作用域

# 系统编程要求
- 多线程同步优先使用互斥锁和条件变量
- 信号处理函数只能调用异步信号安全的函数
- 所有文件描述符必须在使用后关闭
- 避免在信号处理函数中执行耗时操作

# 输出格式要求
1. 首先明确列出本次任务的**可验证成功标准**
2. 说明本次修改涉及的文件和代码范围
3. 只提供修改后的代码片段，使用diff格式标注变更
4. 在关键处和易错处添加调试打印代码
5. 提供完整的编译命令和测试步骤
6. 列出可能的风险点和需要我确认的重大决策

# 特殊要求
- 当我描述的需求不明确时，不要猜测，直接列出需要我澄清的问题
- 当有多种实现方案时，列出优缺点对比，让我选择
- 当需要添加新功能时，先确认功能需求和接口定义，再开始编写
- 测试不通过时，自动分析日志，尝试修复，最多尝试3次后向我报告
